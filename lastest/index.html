Powered by · Google

A lightweight, zero-dependency language selector widget with built-in Google Translate integration. Drop a single <div> into your page — no frameworks, no build step required.

Made by FOXDIM  ·  www.foxdim.com  ·  Translation powered by Google Translate

Installation

Include the script anywhere in your HTML — before </body> is recommended. The widget initializes automatically on DOMContentLoaded.

<!-- Minified (recommended for production) -->
<script src="https://cdn.foxdim.com/repo/foxdim/libs/translate/lastest/translator.min.js"></script>
✅
No other dependencies are required. jQuery, React, Vue — none are needed.
🌐
Set <html lang="..."> correctly (e.g. en, tr, de) for proper translation behavior; if missing/invalid, the library falls back to English (en).

Quick Start

Place a <div> with data-type="foxLang" wherever you want the selector to appear.

<div
  data-type="foxLang"
  data-value="TR,EN,DE"
  data-class="foxLang"
></div>

That's all. The widget renders a button with the current language flag + name. Clicking it opens a searchable modal with all supported languages.

ℹ️
The last selected language is saved in localStorage and restored on the next page load automatically.
🌐
In the language modal, the original document language is placed at the top and marked as Original Language.

Attributes

All configuration is done through HTML data- attributes on the widget element.

AttributeRequiredDefaultDescription
data-type="foxLang"Yes—Marks the element as a FoxLang widget. Required for auto-initialization.
data-valueYestr,enComma-separated list of language codes shown in the "Quick Select" section of the modal (e.g. TR,EN,DE,FR).
data-classYesfoxLangCSS class prefix for all widget elements and the localStorage key prefix. Multiple widgets with the same value share one modal and stay in sync.
data-themeNoautoWidget color scheme: light, dark, or auto. In auto mode the theme is read from the page and updates live.
data-selectoptionNoflag,nameComma-separated tokens controlling what appears on the closed selector button (e.g. flag,iso2,name,en-name). See Display Tokens.
data-listoptionNoflag,name,en-nameComma-separated tokens controlling what appears in each modal list row. See Display Tokens.
data-optionNo—Comma-separated behaviour flags. See Options.

Display Tokens

Tokens define what is rendered in the selector button (data-selectoption) and in each row of the modal list (data-listoption). They are rendered in the order you specify.

flag
Country/language flag SVG icon
iso2
2-letter code badge (e.g. TR, EN)
name
Native name (e.g. Türkçe, Deutsch)
en-name
English name in parentheses (e.g. (Turkish))

Token examples

<!-- Selector: show only flag -->
data-selectoption="flag"

<!-- Selector: compact flag + iso2 badge -->
data-selectoption="flag,iso2"

<!-- Selector: flag + native name -->
data-selectoption="flag,name"

<!-- Selector: full detail — tüm token'lar -->
data-selectoption="flag,iso2,name,en-name"

<!-- Modal list: full info -->
data-listoption="flag,iso2,name,en-name"

<!-- Modal list: minimal -->
data-listoption="flag,name"
💡
When data-selectoption is exactly flag,iso2, the selector button automatically switches to a compact style with tighter padding — perfect for tight navigation bars.

Options (data-option)

Behaviour flags are set as a comma-separated list in data-option.

FlagDescription
no-flagsHides all flag SVGs everywhere (selector button and modal list). Useful for text-only UIs.
yes-auto-translateOn first load (no saved preference), automatically selects the visitor's browser language if it is a supported language.
yes-first-lang-browserMoves the visitor's browser language to the top of the "Quick Select" group in the modal, regardless of data-value order.
 
<div
  data-type="foxLang"
  data-value="TR,EN,DE,FR"
  data-option="yes-auto-translate,yes-first-lang-browser"
  data-class="foxLang"
></div>

Theming

Set data-theme to control the widget's color scheme.

ValueBehaviour
lightAlways uses the light color scheme.
darkAlways uses the dark color scheme.
auto defaultReads the current theme from the page and reacts live to changes. Detection priority below.

Auto-detection priority

When data-theme="auto", the following sources are checked in order:

html[theme="dark|light"] — custom attribute
html[data-theme="dark|light"] — common SPA convention
html[data-bs-theme="dark|light"] — Bootstrap 5
html.classList — contains dark or light class
prefers-color-scheme — OS/browser system preference
<!-- These are all recognized by auto mode -->
<html theme="dark">
<html data-theme="dark">
<html data-bs-theme="dark">
<html class="dark">
⚡
auto mode uses MutationObserver on <html> and listens to matchMedia changes — the widget theme updates instantly without a page reload.

Shared Modal (Multiple Widgets)

When you place multiple widgets with the same data-class value, they share a single modal and a single localStorage key. Selecting a language in one widget instantly updates all of them.

<!-- Widget in the top navbar -->
<div
  data-type="foxLang"
  data-value="TR,EN,DE"
  data-selectoption="flag,iso2"
  data-class="foxLang"
></div>

<!-- Widget in the sidebar — same data-class, same modal -->
<div
  data-type="foxLang"
  data-value="TR,EN,DE"
  data-selectoption="flag,name"
  data-class="foxLang"
></div>
▶ Live
ℹ️
Each data-class value creates one modal. Widgets sharing the same data-class register render callbacks, so selecting a language updates all of them at the same time. For independent widgets with separate modals and separate localStorage, use different values such as foxLang1 and foxLang2.

Auto Translate (Google Translate)

The library ships with a FoxdimTranslate class that wraps Google Translate. When a language is selected in the widget, FoxdimTranslate.setLang() is called automatically if it is present on the page.

How it works

User clicks a language in the modal.
FoxLangSelect.onChange(lang) fires — calls window.FoxdimTranslate.setLang(lang).
FoxdimTranslate injects the Google Translate script if not already loaded, then triggers the language change.
The Google Translate top-bar is automatically hidden.
// Manual trigger via JS (no widget click needed)
window.FoxdimTranslate.setLang('de');   // Translate page to German
window.FoxdimTranslate.restorePage();    // Restore to original language

JavaScript API

Both classes are exposed on window after DOMContentLoaded.

window.FoxLangSelect

Method / PropertyDescription
.registryObject keyed by data-class prefix. Holds shared state (selected lang, renderers, modal references) for each widget group.
.resolveLang(raw)Normalizes a raw language string (e.g. "EN", "en-US") to the internal key used by the widget.
.resolveTheme(raw)Returns "dark" or "light" by reading the current page theme. Evaluates all detection sources.
.onChange(lang)Called after every language change. Override to add custom behavior.

window.FoxdimTranslate

MethodDescription
.setLang(lang)Translates the page to the given language code. Injects Google Translate if needed.
.restorePage()Removes Google Translate and restores the original page content.
.getPageLang()Returns an object { page_lang, browser_lang } with the current <html lang> and navigator language.
.hideGoogleTopbar()Hides the Google Translate floating topbar. Called automatically on translation.
 
// Custom onChange hook — e.g. send analytics event
window.FoxLangSelect.onChange = function(lang) {
  console.log('Language changed to:', lang);
  gtag('event', 'language_change', { language: lang });
};
